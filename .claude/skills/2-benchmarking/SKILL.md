# Benchmarking

## 핵심 원칙

- **Fact 기반 필수**: WebSearch/WebFetch로 최신 정보만 사용 (추측 금지)
- **최신 정보**: 2024-2025년 정보 우선
- **비교 대상 제한**: 한 번에 최대 2-3개
- **Compact 출력**: 비교표 + 구체적 권장사항
- **Wiki 문서화**: 모든 결과 mpm으로 자동 업로드
- **이모지 사용 금지**: 모든 산출물에서 이모지 금지

---

## 비교 대상

### 기본 (우선순위)
1. **Shopify**: E-commerce 플랫폼 (대시보드 UX, 시각화)
2. **Google Analytics 4**: 데이터 분석 도구 (정확성, 실시간 분석)
3. **Amazon Seller Central**: 마켓플레이스 (매출 분석, 경쟁사 비교)

### 추가 (한국 시장)
4. **쿠팡 Wing / 파트너스**: 로켓배송 데이터
5. **네이버 스마트스토어**: 네이버 검색 연동

*전체 리스트: `references/global_ecommerce_targets.md`*

## 워크플로우

### 1. 요청 분석
```
입력: "Shopify 대시보드와 PRO 리포트 비교"

처리:
1. 비교 대상 추출 (최대 2-3개)
2. 비교 목적 명확화 (UI/UX, 기능, 데이터)
3. 검색 키워드 준비
```

### 2. 정보 수집 (WebSearch/WebFetch)
```
검색 전략:
- "[대상] official documentation 2024"
- "[대상] dashboard screenshots 2024"
- "[대상] vs [경쟁사] comparison 2024"

검증:
- 발행일: 2024-2025년 우선
- 출처: 공식 사이트 > 기술 블로그
- 오래된 정보 발견 시: 경고 표시
```

### 3. 비교 분석 템플릿
```markdown
# [비교 주제] 벤치마킹

작성일: {YYYY-MM-DD}
비교 대상: {대상1}, {대상2}
출처: {URL}

## 1. 핵심 기능 비교

| 항목 | {대상1} | {대상2} | 차이점 |
|------|---------|---------|--------|
| 기능 A | 제공 | 미제공 | {설명} |

## 2. UI/UX 비교

### {대상1}
- 장점: {구체적 사례}
- 단점: {구체적 사례}

## 3. 즉시 실행 가능한 권장사항

1. **단기 (1-2주)**: {구체적 액션}
2. **중기 (1-2개월)**: {구체적 액션}
3. **장기 (3개월+)**: {구체적 액션}

## 4. 참고 자료
- [{대상1} 공식 문서]({URL})
```

### 4. Wiki 문서화 (mpm 연동)
```
자동 업로드:
- pageId: 2800416316
- 제목: "[벤치마킹] {대상} vs PRO 리포트 - {주제} ({날짜})"
```

## 체크리스트

**정보 수집 전**:
- [ ] 비교 대상 2-3개로 제한
- [ ] 비교 목적 명확
- [ ] 검색 키워드에 "2024" 포함

**정보 수집 중**:
- [ ] WebSearch로 최신 정보 확인
- [ ] 발행일 2024-2025년
- [ ] 출처 신뢰성 확인

**분석 및 출력**:
- [ ] 비교표에 구체적 정보 (추측 없음)
- [ ] 즉시 실행 가능한 권장사항
- [ ] 참고 자료 URL 명시

**Wiki 문서화**:
- [ ] mpm으로 자동 업로드
- [ ] 날짜 포함

## 출력 예시

**Good (구체적)**:
```markdown
- **실시간 매출 추적**: 5분 단위 업데이트, 모바일 푸시 알림 지원
- **자동 인사이트**: "지난주 대비 전환율 15% 하락" 자동 감지
- **액션 제안**: "할인 프로모션 실행" 원클릭 캠페인 생성
```

**Bad (추상적, 금지)**:
```markdown
- Shopify는 우수한 분석 기능 제공
- 사용자 경험이 좋음
```
