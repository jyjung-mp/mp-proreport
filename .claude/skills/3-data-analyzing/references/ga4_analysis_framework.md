# GA4 분석 프레임워크

## GA4 핵심 철학

### 1. Event-Based (이벤트 기반)
모든 인터랙션 = 이벤트

**PRO 적용:**
- `purchase`, `add_to_cart`, `pro_feature_used`

### 2. User-Centric (사용자 중심)
사용자 여정 중심

**핵심 지표:**
- User Lifetime Value (LTV)
- User Engagement (참여도)
- User Retention (재방문율)

### 3. Predictive (예측 지표)
과거 데이터 기반 미래 예측

**PRO 적용:**
- 성장 가능성 = 현재 성장률 + 업종 평균
- 이탈 위험도 = PRO 미사용 기간

### 4. Attribution (기여도 분석)
다중 채널/기능 기여도 측정

**GA4 모델:**
- Data-driven (데이터 기반)
- Last-click (마지막)
- First-click (첫 번째)
- Linear (균등)
- Time decay (시간 가중)

**PRO 적용:**
```
PRO_매출 = SEO + CRM + 프로모션 + 채널
중복 방지: 하나의 주문 = 하나의 기여도
우선순위: 채널 > 프로모션 > CRM > SEO
```

### 5. Privacy-First (프라이버시 우선)
- 필수 데이터만 수집
- 익명화/집계
- 개인 식별 정보 제외

## 데이터 검증 체크리스트

### 1. Accuracy (정확성)
- [ ] 데이터 소스 명확
- [ ] 계산식 논리적
- [ ] 단위 일관
- [ ] 반올림 규칙 명시

### 2. Completeness (완전성)
- [ ] 필수 필드 존재
- [ ] NULL/0 처리 정의
- [ ] 예외 상황 대응

### 3. Consistency (일관성)
- [ ] 동일 지표 동일 값
- [ ] 합계 = 부분합
- [ ] 기간별 일관된 기준

### 4. Timeliness (시의성)
- [ ] 업데이트 주기 명확
- [ ] 지연(latency) 고려
- [ ] 실시간/배치 구분

### 5. Interpretability (해석 가능성)
- [ ] 사용자 이해 가능 용어
- [ ] 인사이트 명확
- [ ] 액션 가능

## 대시보드 설계 원칙

### Pyramid Structure (피라미드 구조)
상단에 핵심, 하단으로 상세

```
┌─────────────────────────────┐
│  핵심 카드 (3개)             │
├─────────────────────────────┤
│  아바타 인사이트             │
├─────────────────────────────┤
│  매출 추이                   │
├─────────────────────────────┤
│  PRO 성과                    │
└─────────────────────────────┘
```

### Progressive Disclosure (점진적 공개)
- Level 1: 요약
- Level 2: 상세 (클릭)
- Level 3: 분석 (드릴다운)

### Color Psychology (색상)
- Green: 성장/긍정
- Red: 경고/주의
- Blue: 중립/정보
- Orange: 액션

### Context is King (맥락)
숫자만이 아닌 의미 전달

**Bad:** "매출: 1,234,567원"
**Good:** "지난달 대비 15% 증가한 1,234,567원"
